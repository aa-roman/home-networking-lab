# Default Gateway Misconfiguration

## Incident
User device could not communicate with external subnet.

## Symptoms
- Failed connectivity to remote network
- Local communication functional

## Troubleshooting Process
1. Verified local IP configuration
2. Tested local subnet communication
3. Inspected default gateway configuration
4. Identified incorrect gateway assignment
5. Corrected gateway configuration

## Root Cause
Incorrect default gateway prevented inter-network routing.

## Resolution
Restored correct gateway address.

## Skills Practiced
- Routing troubleshooting
- Gateway diagnostics
- Inter-network communication analysis
