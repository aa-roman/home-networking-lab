## Incident
User device unable to communicate with systems in same department VLAN.

## Symptoms
- Failed communication between local department systems
- Connectivity failure isolated to one workstation

## Troubleshooting Process
1. Verified IP addressing
2. Tested subnet communication
3. Inspected switch port VLAN assignments
4. Identified incorrect VLAN assignment
5. Restored correct VLAN configuration

## Root Cause
Switch port assigned to incorrect VLAN.

## Resolution
Reassigned switch port to correct VLAN.

## Skills Practiced
- VLAN troubleshooting
- Switch diagnostics
- Connectivity analysis
- Network segmentation troubleshooting
